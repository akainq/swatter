# ADR-0004: Деплой — self-hosted, один docker-compose

- **Статус:** Accepted
- **Дата:** 2026-07-02
- **Вопрос:** целевая модель развёртывания и её влияние на архитектуру.

## Контекст

Официальный self-hosted Sentry — ~16 контейнеров (Kafka, ZooKeeper, множество воркеров, Snuba, Relay, Symbolicator...) и известная боль в установке и обновлении. Простота self-hosted установки — одно из главных конкурентных преимуществ, которое мы можем предложить.

## Решение

- Целевая инсталляция MVP — **один `compose.yaml` из 4 контейнеров**: `swatter` (одно Elixir-приложение: ingest + пайплайн + API + отдача SPA), `postgres`, `clickhouse`, `redis`.
- Цель: установка с нуля до принятого события — **≤ 10 минут** на одной машине. Это метрика продукта, проверяется в CI (compose up на чистой VM).
- **Single-tenant по продукту, multi-tenant по модели данных**: `org_id` присутствует во всех сущностях и строках CH с первого дня, квоты моделируются per-org/per-project — но UI управления тенантами, биллинг и жёсткая изоляция в MVP не строятся. SaaS позже — без миграции схемы (отдельный ADR при необходимости).
- Вертикальное масштабирование — основной путь для self-hosted; горизонтальное (BEAM-кластер, шардирование CH) — задокументированный путь для крупных инсталляций, не MVP.
- Обновления: миграции схемы (Ecto + миграции CH) выполняются автоматически при старте новой версии; ломающие изменения схемы CH — только через additive-миграции.

## Альтернативы

- **SaaS multi-tenant с первого дня.** Отклонено: тянет квоты, биллинг, изоляцию и совсем другой операционный контур до того, как продукт доказан.
- **Kubernetes/Helm как основной способ установки.** Отклонено для MVP: поднимает порог входа; Helm-чарт возможен позже поверх тех же образов.

## Последствия

- (+) «Одна команда — и работает» как позиционирование против Sentry self-hosted.
- (+) Ограничение в 4 контейнера дисциплинирует архитектуру: каждое «давайте добавим компонент» проходит через это ограничение (и через ADR).
- (−) Потолок одной ноды: единицы тысяч событий/сек (уточним бенчмарком в M1) — для целевой аудитории self-hosted этого достаточно с запасом.
- (−) Автомиграции при старте требуют аккуратности (backward-compatible схемы между соседними версиями).
